2. Basic Rules

Seperti yang sudah dijelaskan sebelumnya, JavaScript tidak diterjemahkan dari kode yang ditulis manusia menjadi bentuk khusus agar browser bisa memahaminya. Skrip JavaScript dikirim ke browser bersama aset lain seperti markup (HTML), gambar, dan stylesheet (CSS). Browser menafsirkannya persis seperti yang ditulis: sebagai urutan karakter Unicode yang bisa dibaca manusia, dan dibaca dari kiri ke kanan serta dari atas ke bawah.

Saat sebuah interpreter JavaScript menerima skrip, pertama-tama ia melakukan analisis leksikal, yaitu memecah rangkaian panjang karakter dalam skrip menjadi beberapa elemen masukan terpisah berikut:

- Tokens: Bagian penting seperti kata kunci (let, if), nama variabel, angka, dan simbol (+, =).
- Format control characters: Karakter khusus untuk mengatur teks, misalnya tanda akhir baris tertentu.
- Line terminators: Penanda akhir baris seperti \n (enter).
- Comments: Teks catatan yang diabaikan saat dijalankan, misalnya // catatan.
- Whitespace: Ruang kosong, tab, atau spasi yang memisahkan bagian kode agar lebih mudah dibaca.

Hasil dari sebuah skrip tidak akan tetap ada setelah halaman dimuat ulang atau ketika berpindah ke halaman lain, dengan kata lain jika kamu menulis kode JavaScript yang mengubah teks atau warna halaman, perubahan itu hilang ketika halaman direfresh atau kamu pindah ke halaman lain, kecuali kamu menambahkan kode khusus untuk menyimpan data (misalnya memakai localStorage, cookies, atau database)

Secara umum, aplikasi JavaScript tersusun dari statements dan expressions.

2.1. Statements

Sebuah statement adalah satuan instruksi yang terdiri dari satu atau lebih baris kode yang mewakili suatu aksi. misalnya membuat variabel, memanggil fungsi, atau menulis if. Berikut adalah contoh penggunaan statement untuk memberi nilai pada variabel bernama myVariable:
contoh code :

let myVariable = 4;
myVariable;

output :

4

Statement sebaiknya diakhiri dengan tanda titik koma (;). Walaupun tidak selalu wajib karena adanya fitur Automatic Semicolon Insertion (ASI). Namun, ASI hanyalah mekanisme perbaikan error, bukan fitur yang bisa diandalkan. Terlalu bergantung pada ASI dapat menimbulkan ambiguitas dan menyebabkan error yang sulit dilacak. Karena itu, praktik terbaik adalah selalu menuliskan titik koma secara eksplisit di akhir statement.

2.2. Block Statements

Block statement digunakan untuk mengelompokkan beberapa perintah (statements) ke dalam satu kesatuan dengan tanda kurung kurawal { }.

Hal ini memungkinkan kita menulis banyak perintah di tempat yang biasanya hanya memperbolehkan satu perintah. Tanda { } disebut juga curly braces, yang menandai awal dan akhir sebuah blok.
contoh :

if (x === 2) {
  console.log("Nilai x adalah 2");
  alert("Halo!");
  x = x + 5;
}

Bagian dalam { } bisa berisi berbagai perintah, misalnya menampilkan teks, melakukan perhitungan, atau memanggil fungsi.

2.3. Expressions

Expression adalah potongan kode yang menghasilkan sebuah nilai, sehingga bisa digunakan di mana saja nilai dibutuhkan.
contoh :

2 + 2;

output :

4
Ini adalah catatan penting. Misalnya, pastikan selalu menaruh tag <script> di bagian bawah <body>.

Code Block

document.getElementById("demo").innerHTML = "Halo Dunia!";

Output

Halo Dunia!
